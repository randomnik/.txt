#Git

#Step.1
Git 설치 밎 환경설정

>> Github 사이트에서 Settings>Emails에 Block command line pushes that expose my email 칸에 체크된 경우 
   Step.2 마지막에 git push origin master를 입력할 경우
   ! [remote rejected] master -> master (push declined due to email privacy restrictions)
   error: failed to push some refs to 'https://github.com/유저명/리포지토리명.git'
   를 띄우며 커밋이 거부됨

   따라서  Settings>Emails에서 Block command line pushes that expose my email 체크 해제할 것.
   개인정보 유출이 우려된다면 아예 다른 이메일로 변경 후 진행

>> Git 설치
>> Git CMD 실행
>> git --version 입력
>> git config --global user.name "깃허브 유저명" 입력
>> git config --global user.email "깃허브 이메일" 입력
>> 프로젝트 폴더 위치한 드라이브로 이동 
   ex) F: 입력 (폴더가 F드라이브에 위치한 경우)
>> 프로젝트 폴더 이동
   ex) cd test 입력(프로젝트 폴더 명이 test인 경우)
>> git init 입력


#Step.2
VScode에서 Github로 업로드

>> Github에서 repository(리포지토리) 생성
>> Github의 default branch 명은 main으로 설정되어있음
   마지막 git push origin master를 에러 없이 실행하기 위해서는 main을 master로 변경해주어야 함
   Github>Settings>Repositories에서 master로 변경할 것

>> VScode에서 Github에 업로드할 파일이 위치한 폴더 열기
>> 상단바 좌측 7번째 터미널>새 터미널 클릭
   하단에 터미널이 보여지게 됨. 하단의 명령어는 모두 터미널에 입력
>> 좌측 위에서 세번째 아이콘(소스제어) 클릭
>> 리포지토리 초기화(Initialize Repository) 클릭
   리포지토리 초기화는 해당 폴더 내 파일을 git 명령어로 관리할 것으로 선언하는 기능
   반응이 없을 경우 git init 입력
   리포지토리 초기화가 뜨지 않고 바로 좌측 목록에 폴더와 파일이 뜨는 경우 생략 가능함
>> git remote add origin 복사한 깃허브 리포지토리 주소 입력
   ※참고한 블로그에는 git remote origin add로 나와 있으나 이 경우 작동하지 않음. 오타로 추정됨
>> git remote -v 입력
   설정한 리포지토리 주소 확인
>> 업로드 할 파일을 좌측 목록에서 파일 옆 + 버튼을 클릭해 추가
>> 좌측 목록 상단 커밋 메세지 입력란에 커밋 메세지 입력
>> 커밋(체크 버튼) 클릭
>> git push origin master 입력
>> Github에 정상적으로 업로드 되었는지 확인

>> 이후 추가로 커밋할 파일은
   >> 업로드 할 파일을 좌측 목록에서 파일 옆 + 버튼을 클릭해 추가
   >> 좌측 목록 상단 커밋 메세지 입력란에 커밋 메세지 입력
   >> Commit(체크 버튼) 클릭
   >> git push origin master 입력
   >> Github에 정상적으로 업로드 되었는지 확인
   ...만 실행하면 됨


#Step.3
Github에서 데스크탑으로 깃 클로닝

>> 클로닝할 리포지토리에서 code>주소 복사
>> Git bash 실행
>> git 클로닝할 드라이브, 폴더 위치 지정
   ex) F: 입력 (저장할 폴더가 F드라이브에 위치한 경우)
   ex) cd test 입력(저장할 폴더 명이 test인 경우) 
>> git clone 복사한 주소 (리포지토리 전체를 복제할 경우)
   git clone --depth=1 복사한 주소(리포지토리에서 최신 변경 소스코드만 복제할 경우) 입력
